# 展示检索一体模块 (DashboardQuerying) — 交接文档

> 本文档面向后续开发者，说明展示检索一体模块的功能、接口定义、筛选规则、运行调度与联调方式。

---

## 1. 模块定位

该模块是一个**独立服务**，不再挂载到 `MediaCrawler/api`。模块职责包含两部分：

1. **结果展示与实时筛选**：读取 SQLite 中某个 `task` 的初筛通过数据，按平台规则执行关键词实时匹配。
2. **运行参数管理与触发**：提供每日运行参数保存、单次试运行触发、每日定时触发与运行状态查看。

当前阶段默认不落库打标签，筛选结果为实时计算结果。

## 3. 页面功能约定

### 3.1 区块 A（运行参数）

支持参数：
- 抓取关键词 `keywords`
- 时间窗口（小时）`hours_back`
- 抓取上限（每平台）`max_notes`
- 视频大小上限 `max_video_size_mb`
- 视频截图帧数 `num_frames`
- 截图去重阈值 `dedup_threshold`
- OCR 设备 `ocr_device`（`auto/cpu/gpu`）
- OCR worker 数 `ocr_workers`
- 每日触发时间 `daily_time`（`HH:MM`）
- 每日自动运行开关 `daily_enabled`
- 爬虫 headless 开关 `headless`

操作：
- `保存参数`
- `单次试运行`

状态展示：
- 当前状态（`idle/running/error`）
- 最近触发来源（`manual/daily`）
- 最近任务 ID
- 最近运行开始/结束时间
- 退出码
- 最近日志滚动窗口

### 3.2 区块 B（结果展示与筛选）

基础行为：
- 默认显示**最新一次 task** 的内容。
- 可切换历史 task。
- task 下拉显示文案为 `YYYY-MM-DD HH:MM`，若同分钟多次任务显示为 `YYYY-MM-DD HH:MM（第N次）`。

展示行为：
- 未输入筛选词时：显示该 task 全部 `initial_passed=1` 内容。
- 输入筛选词后：显示命中结果。
- 每条结果默认简要卡片展示，点击可展开详情。
- 简要卡片直接提供“跳转原帖”按钮。
- 详情区包含 OCR 实际内容。

统计卡：
- 本次 task 总抓取量
- 时间窗口初筛通过量
- 当前筛选词命中量

## 4. 筛选规则契约

### 4.1 多关键词输入

- 支持逗号（英文/中文）和换行分隔。
- 去重并去除空白词。

### 4.2 平台字段匹配

- `bilibili`：在以下字段检索
  - `title`（标题）
  - `desc`（简介）
  - `ocr_text_joined`（OCR内容）

- `xhs`：在以下字段检索
  - `title`（标题）
  - `desc`（内容）
  - `ocr_text_joined`（OCR结果）
  - `ip_location`（发帖IP位置）

### 4.3 匹配模式

后端支持：
- `or`：任一关键词命中即通过（默认）
- `and`：全部关键词均命中才通过

当前页面默认走 `or`。

### 4.4 命中解释

每条返回：
- `matched_keywords`：命中的关键词列表
- `matched_by`：命中的字段来源列表
- `score`：简单匹配分（命中关键词与字段数的线性分）

## 5. 数据源与查询规则

数据库：`/Users/pan/Desktop/新项目/MediaCrawler/data/media_items.db`

task 列表聚合逻辑：
- 按 `task_id` 分组
- `MIN(created_at)` 作为 task 开始时间
- `COUNT(*)` 作为总抓取量
- `SUM(initial_passed=1)` 作为初筛通过量

结果列表查询逻辑：
- 固定条件：`WHERE task_id = ? AND initial_passed = 1`
- 排序：`publish_time DESC, created_at DESC`
- 默认返回上限：`limit=200`（最大 `1000`）

## 6. API 接口定义

### 6.1 `GET /api/dashboard/tasks`

用途：获取 task 列表与默认最新 task。

返回示例：

```json
{
  "latest_task_id": "E2E-202603030600",
  "tasks": [
    {
      "task_id": "E2E-202603030600",
      "label": "2026-03-03 06:00",
      "started_at": 1772498436,
      "ended_at": 1772498501,
      "started_at_text": "2026-03-03 06:00:36",
      "total_crawled": 20,
      "initial_passed_count": 19
    }
  ]
}
```

### 6.2 `GET /api/dashboard/results`

查询参数：
- `task_id`（可选）
- `keywords`（可选）
- `match_mode`（可选，`or/and`，默认 `or`）
- `limit`（可选，默认 `200`）

返回内容：
- task 基本信息
- 统计信息
- 结果列表（含 `matched_keywords`、`matched_by`、`ocr_text_joined`）

### 6.3 `GET /api/dashboard/run-config`

用途：读取当前运行参数与运行状态。

### 6.4 `POST /api/dashboard/run-config`

用途：保存运行参数。

校验规则要点：
- `hours_back/max_notes/max_video_size_mb/num_frames/ocr_workers` 必须 `>0`
- `dedup_threshold` 必须 `>=0`
- `ocr_device` 必须为 `auto/cpu/gpu`
- `daily_time` 支持 `HH:MM` 或 `HH:MM:SS`，入库统一标准化为 `HH:MM`

### 6.5 `POST /api/dashboard/run-once`

用途：触发一次完整流水线（Crawler -> Screenshot -> OCR）。

行为：
- 若已有运行中的任务，返回 `409`。

### 6.6 `GET /api/dashboard/run-status`

用途：获取当前运行状态与最近日志。

## 7. 调度与运行行为

调度机制：
- 服务启动时自动启动轻量 scheduler 循环。
- 每 20 秒轮询一次。
- 当 `daily_enabled=true` 且当前时间到达 `daily_time`，且当日未跑过时，触发 daily run。
- 触发后写入 `last_daily_run_date` 防止同一天重复触发。

运行入口：
- 调度器/单次运行最终都调用：
  - `python 测试模块连贯运行.py --keywords ... --hours-back ... --max-notes ... --max-video-size-mb ... --num-frames ... --threshold ... --ocr-device ... --ocr-workers ... --headless ...`

配置文件：
- `DashboardQuerying/runtime_config.json`

## 8. 与其他模块边界

- 与爬虫、截图、OCR 模块无代码级耦合，不直接 import 对方模块。
- 通过两种方式集成：
  1. 读取统一 SQLite 数据源展示与筛选。
  2. 通过子进程调用根目录流水线脚本执行一次完整流程。

---

## 9. 已知边界与运维提示

1. 每日无人值守场景中，小红书流程可能受登录态/风控影响导致任务失败；该失败会以子进程退出码 `!=0` 体现。  
2. 当前状态面板中的 `last_task_id` 来自运行日志解析，若日志格式变化会影响展示准确性。  
3. 当前日志为内存窗口（最近 400 条），服务重启后历史运行日志不保留。  
4. 当前为实时筛选，不将命中结果持久化写回数据库。
