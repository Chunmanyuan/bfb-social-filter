# 社媒筛选器

一个面向 Windows 的本地工具，用于抓取社媒内容、提取截图与 OCR 文字，并在展示页面中进行检索与筛选。

## 界面预览

![展示检索一体面板](DashboardQuerying/界面图片.png)

## 主要功能

- 支持小红书、B 站等平台的内容抓取
- 支持视频截图与图片文字 OCR 提取
- 支持本地展示页面检索、筛选与查看详情
- 提供面向普通用户的安装引导和启动入口

## 适用范围

- 当前以 Windows 环境为主
- 当前仓库主要用于学习、研究、分享和版本管理
- 普通用户建议使用打包后的程序，不直接运行源码

## 普通用户使用方式

首次使用：

1. 运行 `运行前安装向导.exe`
2. 按提示完成环境安装
3. 登录小红书和 B 站账号

日常使用：

1. 运行 `开始运行.exe`
2. 浏览器会自动打开本地展示页面
3. 在页面中执行检索、单次运行和结果查看

## 界面参数说明

这个界面左侧主要负责设置抓取和处理参数，右侧主要负责结果展示与筛选。

### 左侧参数区

#### 1. 检索关键词

- 用来定义本次要抓取的检索主题
- 可以输入一个或多个词，多个词用逗号分隔
- 例如：`BFB`、`大数据求偶`等

#### 2. 时间窗口（小时）

- 表示系统只抓取“最近多少小时内”的内容
- 例如填 `24`，就是只看最近 24 小时内的新内容

#### 3. 抓取上限（每平台）

- 表示每个平台最多抓多少条符合检索词和时间的内容
- 对于每天更新的bfb内容，建议默认30就足够使用

#### 4. 视频大小上限（MB）

- 表示只处理不超过这个大小的视频文件
- 设置这个上限，主要是为了避免下载和处理过大的视频
- 通常每日bfb内容保持默认的100mb设置即可

这样做的原因：

- 通常过大的视频并非bfb内容而是包含bfb标签的其他内容（例如恩爱日常、旅行vlog等）
- 过大的视频会大量占用空间


#### 5. 视频截图帧数

- 表示从一个视频里抽取多少帧用于后续筛查和 OCR
- 数值越大，提取信息越充分
- 但数值越大，处理时间也会更长
- 通常每日bfb内容保持默认的40设置即可

#### 6. 截图去重阈值

- 用于减少重复截图
- 如果视频前后画面非常相近，系统会尽量避免保存太多重复帧
- 这个参数主要影响截图结果的精简程度

#### 7. OCR workers

- 表示 OCR 同时使用多少个处理进程
- 一般电脑配置普通时，保持 `1` 就比较稳
- 配置较高时可以适当调大，但不建议盲目提高

#### 8. OCR 设备

- 暂时仅支持CPU运行，若想使用GPU加速运行请自行代码改动

#### 9. 每日触发时间

- 表示每天定时运行的时间点
- 例如 `06:00` 就表示每天早上 6 点自动执行一次

#### 10. 每日自动运行

- 勾选后，系统会按“每日触发时间”定时执行
- 不勾选时，只会在你手动点击时运行

#### 11. 频繁失败登录时开启此选项

- 平时默认不勾选
- 如果遇到平台频繁登录失败，可以勾选这个选项
- 勾选后会弹出 Chrome 窗口，便于重新登录

### 右侧结果区

#### 1. 输入筛选词

- 用于在本次结果里进一步筛选
- 支持输入一个或多个词
- 多个词可用逗号或换行分隔
- 例如：`上海`、`成都`

#### 2. 执行筛选

- 点击后会在当前结果中筛选命中的字段和命中的词

#### 3. 三个统计框

- `本次 Task 总抓取量`：本次任务抓到了多少条内容
- `时间窗口初筛通过量`：在时间窗口内通过初筛的数量
- `关键词命中量`：在结果中命中筛选词的数量

## 使用案例

下面给一个比较容易理解的案例：

### 目标：每日6点抓取上海和成都的bfb内容

- 检索关键词：`BFB`
- 每天早上 `06:00` 自动运行一次
- 抓取后自动进行 OCR
- 在结果页面里再筛选 `上海` 或 `成都`

### 建议设置

左侧参数可这样填写：

- 抓取关键词：`BFB`
- 时间窗口（小时）：`24`
- 抓取上限（每平台）：`30`
- 视频大小上限（MB）：`100`
- 视频截图帧数：`40`
- 截图去重阈值：`5`
- OCR workers：`1`
- OCR 设备：`auto`
- 每日触发时间：`06:00`
- 每日自动运行：勾选

右侧筛选词可填写：

- `上海`
- 或 `成都`
- 也可以填写成 `上海, 成都`

### 这个案例的实际含义

- 系统每天早上 6 点自动执行一次抓取
- 只关注最近 24 小时内与 `BFB` 相关的内容
- 抓取后会对图片和视频截图进行 OCR
- 最后在展示页面中，再进一步查看是否命中了 `上海` 或 `成都`

## 开发目录说明

- `DashboardQuerying/`：展示检索服务与前端页面
- `MediaCrawler/`：社媒抓取相关逻辑
- `VideoScreenshotter/`：视频截图处理
- `PaddleOCRProcessor/`：OCR 文字提取
- `Launcher/`：绿色版启动器与安装引导

## 本地开发运行

1. 准备 Python 环境
2. 安装 `requirements_all.txt` 中的依赖
3. 启动 `DashboardQuerying/app.py`
